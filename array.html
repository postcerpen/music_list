Introduction

This guide covers various data storage and manipulation techniques in Godot Engine. Understanding these concepts is crucial for game development, as you'll need to store player data, game states, settings, and more.

We'll explore arrays, dictionaries, JSON, file operations, and other storage methods with practical examples.

Arrays

Arrays are ordered collections of values. In Godot, you can create arrays using the Array class or the shorthand syntax.

Creating Arrays

# Empty array
var my_array = []

# Array with values
var numbers = [1, 2, 3, 4, 5]
var strings = ["apple", "banana", "cherry"]
var mixed = [1, "hello", true, 3.14]

# Using Array constructor
var arr = Array()

Common Array Operations

var fruits = ["apple", "banana"]

# Add elements
fruits.append("cherry")  # ["apple", "banana", "cherry"]
fruits.push_back("date") # ["apple", "banana", "cherry", "date"]

# Remove elements
fruits.remove(1)  # Remove at index 1: ["apple", "cherry", "date"]
var last = fruits.pop_back()  # last = "date", fruits = ["apple", "cherry"]

# Access elements
var first = fruits[0]  # "apple"
fruits[1] = "blueberry"  # ["apple", "blueberry"]

# Array size
var size = fruits.size()  # 2

# Check if array contains value
var has_apple = fruits.has("apple")  # true

# Iterate through array
for fruit in fruits:
    print(fruit)

Dictionaries

Dictionaries are collections of key-value pairs. They're similar to objects in JavaScript or dictionaries in Python.

Creating Dictionaries

# Empty dictionary
var my_dict = {}

# Dictionary with values
var player = {
    "name": "John",
    "health": 100,
    "level": 5,
    "inventory": ["sword", "potion"]
}

# Using Dictionary constructor
var dict = Dictionary()

Common Dictionary Operations

var character = {
    "name": "Alice",
    "health": 100
}

# Add/update values
character["level"] = 3
character.erase("health")  # Remove key

# Access values
var name = character["name"]
var level = character.get("level", 1)  # Get with default value

# Check if key exists
if character.has("name"):
    print("Has name key")

# Dictionary size
var size = character.size()

# Get all keys or values
var keys = character.keys()
var values = character.values()

# Iterate through dictionary
for key in character:
    print(key + ": " + str(character[key]))

Working with JSON

JSON (JavaScript Object Notation) is a lightweight data interchange format. Godot has built-in support for parsing and generating JSON.

Parsing JSON

# JSON string to Dictionary
var json_string = '{"name": "John", "age": 30, "city": "New York"}'
var json = JSON()
var parse_result = json.parse(json_string)

if parse_result == OK:
    var data = json.get_data()
    print(data["name"])  # "John"
else:
    print("JSON parse error")

Generating JSON

# Dictionary to JSON string
var data = {
    "player_name": "Alice",
    "score": 1500,
    "items": ["sword", "shield"]
}

var json_string = JSON().stringify(data)
print(json_string)
# Output: {"player_name":"Alice","score":1500,"items":["sword","shield"]}

func save_game():

File Storage

Godot provides several ways to work with files for data persistence.

Basic File Operations

# Write to file
func save_to_file(path, content):
    var file = FileAccess.open(path, FileAccess.WRITE)
    file.store_string(content)
    file.close()

# Read from file
func read_from_file(path):
    var file = FileAccess.open(path, FileAccess.READ)
    if file:
        var content = file.get_as_text()
        file.close()
        return content
    return ""

# Check if file exists
func file_exists(path):
    return FileAccess.file_exists(path)

Special File Paths

Path Description
"res://" Read-only resource path (included with game)
"user://" User data path (for saves, settings, etc.)
"user://saves/" Good practice to organize saves in a folder

Working with Directories

# Create directory
var dir = DirAccess.open("user://saves")
if !dir:
    dir = DirAccess.make_dir_recursive_absolute("user://saves")

# List files in directory
func list_save_files():
    var saves = []
    var dir = DirAccess.open("user://saves")
    if dir:
        dir.list_dir_begin()
        var file_name = dir.get_next()
        while file_name != "":
            if !dir.current_is_dir() and file_name.ends_with(".json"):
                saves.append(file_name)
            file_name = dir.get_next()
    return saves

Config Files

Godot's ConfigFile class provides an easy way to store settings and configuration data.

Using ConfigFile

# Save settings
func save_settings():
    var config = ConfigFile()
    
    # Store values
    config.set_value("audio", "master_volume", 0.8)
    config.set_value("audio", "music_volume", 0.6)
    config.set_value("graphics", "fullscreen", true)
    config.set_value("gameplay", "difficulty", "normal")
    
    # Save to file
    config.save("user://settings.cfg")

# Load settings
func load_settings():
    var config = ConfigFile()
    var error = config.load("user://settings.cfg")
    
    if error == OK:
        # Get values with defaults
        master_volume = config.get_value("audio", "master_volume", 1.0)
        music_volume = config.get_value("audio", "music_volume", 1.0)
        fullscreen = config.get_value("graphics", "fullscreen", false)
        difficulty = config.get_value("gameplay", "difficulty", "easy")

Custom Resources

Godot's Resource system allows you to create custom data types that can be saved and loaded like built-in resources.

Creating a Custom Resource

# player_data.gd
extends Resource
class_name PlayerData

@export var player_name: String = ""
@export var level: int = 1
@export var experience: int = 0
@export var inventory: Array[String] = []

Using Custom Resources

# Create and save resource
func save_player_data():
    var player_data = PlayerData.new()
    player_data.player_name = "Hero"
    player_data.level = 5
    player_data.experience = 1250
    player_data.inventory = ["sword", "potion", "key"]
    
    ResourceSaver.save(player_data, "user://player_data.tres")

# Load resource
func load_player_data():
    if ResourceLoader.exists("user://player_data.tres"):
        var player_data = ResourceLoader.load("user://player_data.tres")
        print(player_data.player_name)  # "Hero"
        print(player_data.level)        # 5

Best Practices

Error Handling

# Always check for errors when working with files
func safe_save(path, data):
    var file = FileAccess.open(path, FileAccess.WRITE)
    if file:
        file.store_string(data)
        file.close()
        return true
    else:
        push_error("Failed to save file: " + path)
        return false

Data Validation

# Validate data before saving
func validate_player_data(data):
    if !data.has("name") or data["name"] == "":
        return false
    if !data.has("health") or data["health"] < 0:
        return false
    return true

Backup System

# Create backup before saving
func safe_save_with_backup(path, data):
    # Create backup if file exists
    if FileAccess.file_exists(path):
        var backup_path = path + ".backup"
        DirAccess.rename_absolute(path, backup_path)
    
    # Save new file
    return safe_save(path, data)
